Labeling Policy

How the paper tables turn raw analyzer rule ids into reviewer-readable labels.

  • Main paper tables use a human-readable concept label first, not a raw analyzer id.
  • Equivalent raw rules are merged when they represent the same concept across analyzers or language variants. For example, no-unused-vars and @typescript-eslint/no-unused-vars are shown as a single concept.
  • Raw rule names and ids are preserved on this page only, so reviewers can audit exactly which analyzer rules are included.
  • Issue families are derived semantically: Security for security analyzer findings, Bug for correctness/runtime bug rules, and Code Smell for the remaining actionable maintainability findings.
